# Next Steps For Sumit

Your Supabase database is now created. You also have GitHub and Vercel accounts. The next step is putting the production app files into GitHub and importing that GitHub project into Vercel.

## What To Copy From Supabase

Open Supabase and go to:

Project Settings -> Data API

Copy only these two values:

1. Project URL
2. publishable key

Do not copy or share the service_role key.

## What Happens Next

Once the app is in GitHub:

1. Vercel can publish it online.
2. Vercel can safely store the Supabase URL and publishable key.
3. We can test the live site.
4. Then we can wire login and real database saving.

## Plain English

Supabase is the online filing cabinet.
GitHub is where the app files live.
Vercel puts the app online.
The Project URL and publishable key tell the app which Supabase filing cabinet to use.
